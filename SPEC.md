# tomuroboxmidi 実装仕様書

## 1. 概要

`tomuroboxmidi` は、任意の MIDI ファイルを Muro Box 向け形式に変換する Python 製 CLI ツールです。
以下の変換処理を行います。

- 複数トラックを単一トラックにマージ
- 対象モデルの音域外のノートを削除
- 発音タイミングが完全に重複するノートを削除

---

## 2. コマンド仕様

### 基本構文

```
tomuroboxmidi [OPTIONS] <input_file> [<input_file> ...]
```

### 引数

| 引数 | 説明 |
|---|---|
| `input_file` | 変換対象の MIDI ファイルパス（1 つ以上必須、glob 未展開のためシェルに任せる）|

### オプション

| オプション | 説明 |
|---|---|
| `-f`, `--force` | 出力先に同名ファイルが存在しても確認なしに上書きする |
| `-o <dir>`, `--output <dir>` | 出力先ディレクトリを指定（絶対パス・相対パス両方可）。省略時は実行ディレクトリ直下の `muroboxmidi/` |
| `-m <model>`, `--model <model>` | 対象の Muro Box モデルを指定。`n20` または `n40`（デフォルト）|

### 使用例

```sh
# カレントディレクトリの muroboxmidi/ に出力
tomuroboxmidi song.mid

# 複数ファイルを一括変換
tomuroboxmidi *.mid

# 出力先を指定して強制上書き
tomuroboxmidi -f -o ./out song.mid

# n20 モデル向けに変換（デフォルトは n40）
tomuroboxmidi -m n20 song.mid
```

---

## 3. 出力先の決定ロジック

1. `-o` オプションが指定された場合、そのパスを出力ディレクトリとする（相対パスは実行ディレクトリ基準で解決）。
2. 省略された場合、実行ディレクトリ直下の `muroboxmidi/` を出力ディレクトリとする。
3. 出力ディレクトリが存在しない場合、自動的に作成する。
4. 出力ファイル名は入力ファイルのベースネーム（ファイル名部分のみ）をそのまま使用する。

---

## 4. 上書き確認ロジック

1. 出力ファイルが既に存在し、かつ `-f` オプションが指定されていない場合、ユーザーに確認を求める。
2. 確認プロンプト例:

   ```
   出力先ファイルが既に存在します: ./muroboxmidi/song.mid
   上書きしますか？ [y/N]:
   ```

3. `y` または `Y` が入力された場合のみ上書きする。それ以外（`N`、Enter のみ、など）の場合はそのファイルをスキップし、次のファイルへ進む。
4. 複数ファイルを処理する場合は各ファイルごとに個別に確認する。
5. `-f` が指定されている場合は確認なしに全ファイルを上書きする。

---

## 5. MIDI 変換処理

処理は以下の順序で行う。

### 5-1. トラックのマージ

- `mido.merge_tracks()` を使用して全トラックを単一トラックに統合する。
- タイムベース（`ticks_per_beat`）は元ファイルのものを引き継ぐ。
- 出力は Type 0（シングルトラック）MIDI ファイルとする。
- マージ後のトラック名は元ファイルの最初のトラック名、または `Merged` とする。

### 5-2. 音域外ノートの削除

- `note_on` / `note_off` メッセージのうち、対象モデルの有効ノートセット（後述）に含まれないピッチのものを削除する。
- `note_on` を削除した場合、対応する `note_off`（同一ピッチ）も必ず削除する。
- `note_off` を削除した場合、対応する `note_on` も必ず削除する。
- ノートを削除した際は、そのメッセージの `time`（tick）を次のメッセージに加算してタイミングがずれないようにする。
- 削除したノートの数は変換サマリーとして標準出力に表示する。

### 5-3. 重複ノートの削除

- 「発音タイミングが完全に重複するノート」とは、**絶対 tick 位置と pitch の両方が同一の `note_on`（velocity > 0）** が複数存在する場合を指す。
- 重複する場合は最初の 1 件を残し、それ以降を削除する。
- 対応する `note_off` も同様に処理する。
- 削除したノートの数は変換サマリーとして表示する。

## 6. Muro Box 音域仕様

### ノート番号の基準

本ツールでは **C3 = MIDI ノート番号 60**（Middle C, 261.6 Hz）を採用する。

### n20 モデルの有効ノートセット

音域: C3 〜 A5（白鍵のみ、全 20 音）

| ノート名 | MIDI # | ノート名 | MIDI # |
|---|---|---|---|
| C3 | 60 | C4 | 72 |
| D3 | 62 | D4 | 74 |
| E3 | 64 | E4 | 76 |
| F3 | 65 | F4 | 77 |
| G3 | 67 | G4 | 79 |
| A3 | 69 | A4 | 81 |
| B3 | 71 | B4 | 83 |
| | | C5 | 84 |
| | | D5 | 86 |
| | | E5 | 88 |
| | | F5 | 89 |
| | | G5 | 91 |
| | | A5 | 93 |

有効ノート番号セット（Python の `set` として）:

```python
n20_VALID = {60, 62, 64, 65, 67, 69, 71,
             72, 74, 76, 77, 79, 81, 83,
             84, 86, 88, 89, 91, 93}
```

### n40 モデルの有効ノートセット

音域: F2 〜 C6（クロマチック全音、ただし下記 4 音を除く）

除外ノート:
- F#2 (MIDI 54)
- G#2 (MIDI 56)
- A#2 (MIDI 58)
- C#3 (MIDI 61)

有効ノート番号セット（Python の `set` として）:

```python
n40_VALID = set(range(53, 106)) - {54, 56, 58, 61}
# 53 = F2, 96 = C6
```

## 7. エラーハンドリング

| ケース | 挙動 |
|---|---|
| 入力ファイルが存在しない | エラーメッセージを表示してそのファイルをスキップし次のファイルへ進む |
| 入力ファイルが MIDI ファイルとして読み込めない | エラーメッセージを表示してスキップ |
| 出力ディレクトリを作成できない（権限エラー等）| エラーメッセージを表示してツール全体を終了（exit code 1） |
| 入力ファイルが 0 件 | 使用方法を表示して終了（exit code 1） |
| 変換後にノートが 0 件になる | 警告を表示した上でファイルは出力する |

---

## 8. 出力メッセージ仕様

各ファイルの変換後に以下のサマリーを標準出力に表示する。

```
[変換完了] song.mid -> ./muroboxmidi/song.mid
  - 音域外削除: 5 ノート
  - 重複削除: 2 ノート
  - 残存ノート: 142 ノート
```

全処理終了後に集計を表示する。

```
処理完了: 3 ファイル中 3 ファイル変換 (スキップ: 0)
```

## 9. 実装構成

### ファイル構成

```
tomuroboxmidi/
├── main.py              # エントリポイント（CLI 引数パース・処理フロー制御）
├── converter.py         # MIDI 変換ロジック本体
├── models.py            # モデル別有効ノートセット定義
├── pyproject.toml
└── SPEC.md
```

### 各モジュールの責務

#### `main.py`

- `argparse` を用いて `-f`、`-o`、`-m` オプションと入力ファイル引数をパース
- 出力ディレクトリの解決・作成
- 各入力ファイルに対してループし、上書き確認 → `converter.convert()` の呼び出し → サマリー表示

#### `converter.py`

主要関数:

```python
def convert(
    input_path: Path,
    output_path: Path,
    valid_notes: set[int],
) -> ConvertResult:
    ...

# ConvertResult: dataclass
# - removed_out_of_range: int  (音域外削除数)
# - removed_duplicates: int    (重複削除数)
# - remaining_notes: int       (残存ノート数)
```

処理フロー:

1. `mido.MidiFile(input_path)` で読み込み
2. `mido.merge_tracks(mid.tracks)` でトラックマージ
3. 音域外ノートの削除（`_remove_out_of_range()`）
4. 重複ノートの削除（`_remove_duplicates()`）
5. `MidiFile(type=0, ticks_per_beat=..., tracks=[merged_track])` で Type 0 ファイル生成
6. `mid.save(output_path)` で保存

#### `models.py`

```python
n20_VALID: frozenset[int] = frozenset({60, 62, 64, 65, 67, 69, 71,
                                        72, 74, 76, 77, 79, 81, 83,
                                        84, 86, 88, 89, 91, 93})

n40_VALID: frozenset[int] = frozenset(set(range(53, 106)) - {54, 56, 58, 61})

MODEL_MAP: dict[str, frozenset[int]] = {
    "n20": n20_VALID,
    "n40": n40_VALID,
}
```
